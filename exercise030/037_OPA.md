1. Create ConstraintTemplate
The template defines the structure for required labels. This template can be reused for different constraints

```
nano opaconstemp.yaml
```

```
apiVersion: templates.gatekeeper.sh/v1
kind: ConstraintTemplate
metadata:
  name: k8srequiredlabels
spec:
  crd:
    spec:
      names:
        kind: K8sRequiredLabels
      validation:
        openAPIV3Schema:
          type: object
          properties:
            labels:
              type: array
              items:
                type: string
              description: "List of required label keys"
  targets:
    - target: admission.k8s.gatekeeper.sh
      rego: |
        package k8srequiredlabels

        # Check each required label
        violation[{"msg": msg}] {
          # Get the list of required labels from parameters
          required := input.parameters.labels[_]
          # Check if the label exists on the resource
          not input.review.object.metadata.labels[required]
          # Build a descriptive error message
          msg := sprintf("Resource '%v' is missing required label: '%v'", [
            input.review.object.metadata.name,
            required
          ])
        }
```

```
kubectl apply -f opaconstemp.yaml
```

```
kubectl get constrainttemplates.templates.gatekeeper.sh 
```

2. Create the Constraint
The constraint specifies the target resources (Pods) and the namespace scope using namespaceSelector.

```
kubectl create ns opans
```

```
cat << 'EOF' > labels.yaml
apiVersion: constraints.gatekeeper.sh/v1beta1
kind: K8sRequiredLabels
metadata:
  name: require-app-team-labels
spec:
  # enforcement action: deny, dryrun, or warn
  enforcementAction: deny
  match:
    kinds:
      - apiGroups: [""]
        kinds: ["Pod"]
    namespaces:
      - opans
  parameters:
    labels:
      - "env"
EOF
```

```
kubectl apply -f labels.yaml
```

OPA automatically creates new CRDs with its templates.
```
kubectl get k8srequiredlabels.constraints.gatekeeper.sh
```

```
kubectl api-resources | grep required
```

If we try to create a pod without the right label it will be rejected.
```
cat << 'EOF' > opanot.yaml
apiVersion: v1
kind: Pod
metadata:
  name: notallowed
  namespace: opans
spec:
  containers:
  - image: nginx
    name: notallowed
EOF
```

```
kubectl apply -f opanot.yaml
```

Adding the "test" label allows us to create the Pod
```
cat << 'EOF' > opaallowed.yaml
apiVersion: v1
kind: Pod
metadata:
  labels:
    env: test
  name: notallowed
  namespace: opans
spec:
  containers:
  - image: nginx
    name: notallowed
EOF
```

```
kubectl apply -f opaallowed.yaml
```
